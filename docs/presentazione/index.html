WebinarProposta tecnica

Il problema

Serve un form modificabile dal Marketing, ma l'integrazione API deve restare sotto controllo.

Autonomia

Marketing deve poter cambiare domande, testi, campi obbligatori e consenso senza deploy o ticket tecnici.

Segreti API

Il token Magnews non puo' stare nella pagina GitHub Pages o in JavaScript pubblico.

Qualita' dati

Ogni invio deve essere validato, tracciato e mandato alla lista corretta in Magnews.

La pagina web raccoglie interesse. Il backend sicuro gestisce l'integrazione.

WebinarObiettivo

Cosa evitare

GitHub Pages non deve chiamare Magnews direttamente.

Form statico diretto

  • Il token API finisce nel browser.
  • Chiunque puo' leggere e riusare la credenziale.
  • Spam, retry e log restano difficili da controllare.
VS

Ponte server-side

  • Il token resta in un ambiente privato.
  • La chiamata Magnews parte da codice controllato.
  • Possiamo validare dati, consenso e duplicati.
WebinarRischio principale

Scelta consigliata

Google Form + Apps Script e' il compromesso migliore per questo caso.

Perche' Google Form

Google fornisce editor visuale, hosting del form, raccolta risposte e collegamento con Google Sheet. Marketing lavora in uno strumento gia' comprensibile.

Perche' Apps Script

Lo script gira lato server su Google, riceve l'evento di invio e chiama Magnews senza pubblicare il token nel sito.

WebinarSchema architetturale

Sicurezza

Le protezioni minime da mettere nel pilota.

  • Token Magnews in Script Properties, mai nel foglio, nel sito o nel codice pubblico.
  • Permessi separati: Marketing modifica il form; dev/admin modifica Apps Script e credenziali.
  • API user dedicato con permessi minimi per contatti e audience.
  • Consenso esplicito con testo, data, sorgente e finalita' della raccolta.
  • Log sanitizzati: stato e request id, non token o dati personali completi.
  • Validazione input: email, campi obbligatori e mapping stabile verso Magnews.
  • Duplicati e disiscritti: verificare il comportamento Magnews prima del go-live.
  • Retry controllato: gestione errori 401, 403, 429 e 5xx senza invii infiniti.
